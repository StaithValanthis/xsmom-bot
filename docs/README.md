## üõ†Ô∏è Keeping Documentation Updated

### Auto-Generated Docs

Some documentation is auto-generated from the codebase:

- **Module Map** (`docs/architecture/module_map.md`)
- **Config Reference** (`docs/reference/config_reference.md`)

**To regenerate:**

```bash
python -m tools.update_kb
```

**When to run:**
- After adding new modules to `src/`
- After modifying `config/config.yaml.example`
- Before committing major changes (consider pre-commit hook)

### Manual Updates

The following docs should be updated manually when the framework changes:

- `kb/framework_overview.md` - Major architectural changes
- `kb/change_log_architecture.md` - Framework-level change history
- `architecture/strategy_logic.md` - Strategy changes
